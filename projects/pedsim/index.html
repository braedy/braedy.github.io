PedSim The visual output showing two groups of pedestrians moving towards contrasting destinations.
Calculations on movement are performed through different algorithms using the parallel frameworks.

  • Technologies: C++, OpenCL, SIMD / AVX, OpenMP, Pthreads
  • Course: Low-Level Parallel Programming / Lågniva-parallellprogrammering
    @ Uppsala University, Sweden; inc. further personal work.
  • Role: Programmer
  • Year: Spring 2015

This project involved refactoring an existing pedestrian simulation implementation from an open-source library (PedSim).
Refactoring was performed to parallelise the math behind moving each individual (agent) towards its destination. Work focussed on constructing parallel algorithms for the new programming frameworks, identify sections for parallelism and the factors and dependencies affecting its implementation.

I took on the implementation of the OpenCL implementation personally. Due to time constraints a mere basic version was created for submission, but later I went back to see if I could get it working to the level I know it could produce. I focussed namely on how memory was managed between the host and the device. Enqueuing commands to buffers with pointers turned out to be massively faster than allocating buffer reads and writes each pedestrian step cycle.

In addition, we exposed data parallelism through refactorisation of the code, this was especially advantageous for GPU computing and SIMD. Agents in the AVX/OpenCL implementation are structured in a struct of arrays, which contributed to a vast improvement of speed against the Pthread and OpenMP implementations. Each processor performs the same task on different data in parallel. Vectors are passed to the processor where it runs tasks in parallel over different pieces of distributed data.



Since this is an project continually used within university courses, code is only made available upon request.
