This page reflects upon a personal project I am undertaking in order to better my ability programming graphics. I have also started this project to learn DirectX as it has a wider use in the videogame industry than OpenGL.

  • Technologies: DirectX SDK, HLSL, C++, Visual Studio
  • Year: Summer 2015 (Ongoing)

Since recent updates to the SDK, there is yet to be a wealth of good tutorial resources. Therefore much I am learning on my own, basing my implementations on previous DX11 tutorials, however utilising the new libraries (i.e. DirectXMath).

Progress thus far:

  • Direct3D device setup
  • Reading input with DirectInput
  • 3D model rendering
  • Texture mapping
  • Phong shading (diffuse, ambient, specular lighting)
  • 2D graphic rendering
  • Text rendering
  • Frustum culling
  • Multitexture rendering
  • Light maps
  • Projective texturing
  • Projected light maps

    Culling the view frustum
    Models are rendered only when incident with the view frustum. Models escaping the view frustum are discarded. This removes inefficient rendering, which can become expensive if the scene has - for example - 5000 models where only 30 are viewable at a given time. Text rendering, directed lighting and Phong shading has also been implemented.

    Blending multiple textures

    Rendering multiple textures requires a texture array to be passed to the shader, where it is subsequently blended (with an added gamma value). Textures are loaded with a bespoke handler.

    The above image is a blend of a pebble texture and a water surface texture.

    Projected texture rendered onto 3D scene from viewpoint

    A frustum is created from the viewpoint where the texture is to be projected. Where the frustum intersects a 3D object we draw a pixel from the projected texture that matches the projected location.

    Light map projected onto scene

    Projected light maps follows on in the same vein as projective texturing. The implementation is very similar, only requiring an extra buffer to handle the point light. The light map is passed as a texture to the pixel shader.
